Meridian Home Services — AI Contact-Center Assistant

Architecture & request workflow. A document-grounded assistant that answers policy/FAQ questions with citations, runs an agentic ZIP-eligibility → create/reschedule/cancel booking flow with confirm-before-commit, and hands off to a human when unsure, out of scope, or facing an emergency. Built on the three principles it's judged on — sustainability, reproducibility, generalization.

1.The core idea — schema is the API, data is compiled from the documents

No hand-authored business facts. We hand-define only a small, finite set of capability schemas (the shape of a fact — a fee has tiers, coverage has ZIP ranges). Every value (prices, fees, ZIP coverage, hours) is compiled from the documents at build time. At runtime the booking-critical path uses only those compiled records + deterministic code — no LLM or retrieval can change who we book or what we charge. Drop in 1,000 new PDFs, re-run two commands, zero code changes.

2.Request lifecycle — the LangGraph state machine

Every turn flows safety → classify → route → … → confirm → commit → respond, with a handoff escape. The LLM classifies intent, extracts raw slots, and phrases replies; code resolves dates, gates coverage, computes fees, and performs the mutation. Confirm-before-commit is guaranteed by the graph topology (the mutating commit node is reachable only after an approved confirm) + a read-only/mutating capability split + the mock API's independent mutation ledger.

4.Safety & guardrails

🚨 Emergency first

Recall-biased keyword rules run before anything; an LLM paraphrase-union may only add. Straight to the 24/7 line — never a booking.

✋ Confirm before commit

Guaranteed by topology + capability split + per-turn assertion, cross-checked by the API's independent mutation ledger.

🧷 Grounded or abstain

A knowledge answer with no inline citation is not trusted → it hands off rather than state an un-grounded "fact".

🔒 PII / ownership gate

Technician name, notes, invoice are owner-only — withheld unless a matching customer_id is given (then asks to verify).

🛡️ Prompt-injection defense

Retrieved text is fenced/labelled untrusted; the mutating split means a successful injection still can't act; web output is escaped.

💲 Fees never waived by the model

Every figure is computed by code from compiled records; the LLM only phrases it.

5.Evaluation & reproducibility

A functional conformance suite run through the real agent over the in-process Booking double, scored on a trust hierarchy. Temperature-0 LLM calls replay from a committed cache, so results reproduce offline, bit-for-bit, keyless.

① Categorical safety (hard gates)

Emergency recall (never miss / never book) and confirmation-gating (no mutation before approval) — proven against the mutation ledger, zero tolerance.

② Deterministic correctness

Per case: intent · route · kind · action effects (committed?) · citations · fact-substring · PII-not-leaked. Keyless.
